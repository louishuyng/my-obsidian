- Stop: the data on disk (EBS) is kept intact in the next start
- Terminate: any EBS volumnes (root) also set-up to be destroyed is lost

- On start, the following happens:
	- First start: the OS boots & the EC2 User Data script is run
	- Following starts: the OS boots up
	- Then your application starts, caches get warmed up, and that can take time!
- Introducing Hibernate:
	- The in-memory (RAM) state is preserved
	- The instance boot is much faster! (the OS is not stopped / restarted)
	- Under the hood: the RAM state is written to a file in the root EBS volume![[Screen Shot 2022-04-03 at 21.07.28.png]]
	- Use cases
		- long-running processing
		- saving the RAM state
		- services that take time to initialize
	- Supported instance families - C3, C4, C5, M3, M4, M5, R3, R4 and R5
	- Instance RAM size must be less than 150 GB
	- Instance size not supported for bare metal instances
	- AMI: Amzon Linux 2, Linux AMI, Ubuntu, ... & Windows
	- Root Volumne: must be EBS, encrypted, not instance store, and large
	- Available for On-Demand and Reserved Instances
	- An instance can not be hibernated more than 60 days
